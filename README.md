# 가늠터 (Ganeumteo)

> 건축 인허가 AI 사전진단 — 주소와 용도만 입력하면, 그 땅·건물에 원하는 건축행위가 가능한지 LLM 에이전트가 종합 진단합니다.

## 무엇인가

빈 땅에 카페를 신축할 수 있는지, 기존 건물 한 층을 학원·헬스장·유흥주점으로 용도변경할 수 있는지 — 일반인이 건축사 상담 전에 스스로 가늠해볼 수 있도록, **입지·행위제한·조례·의제(함께 처리되는 인허가)·제출서류·규모상한·작성주체**를 한 번에 진단합니다.

- 입력: 주소 + 무엇을(용도) + 어떻게(신축/용도변경/대수선 등)
- 출력: 4등급 종합판정(가능 / 가능(조건부) / 위험·금지 / 확인필요) + 근거 인용 + 단계별 제출서류 로드맵 + 규제효과·부담금 가늠

## 핵심 원칙 — 무하드코딩

케이스별 분기, 서류명, 요율, 가부 판정을 **코드에 박지 않습니다.** LangGraph 기반 ReAct 에이전트가 매 진단마다 공공 API에서 법령·공간 데이터를 라이브로 가져오고, 의미 판단(용도 분류·가부·시설군 이동방향 등)은 LLM이 내립니다. 코드는 라우팅·형식 파싱·집계·법정상수(예: 에너지절약계획서 500㎡, 1평=3.3058㎡)·**fail-closed 안전판정**만 담당합니다.

## 데이터 소스 (전부 라이브)

| 소스 | 용도 |
|---|---|
| **VWorld** | 주소→좌표, 지적(지목·도로접면), 용도지역·규제중첩, 공시지가, 공간레이어 |
| **data.go.kr (1613000)** | 행위제한(토지이용), 건축물대장 표제부·층별개요(건축HUB) |
| **law.go.kr (DRF)** | 법령·자치법규 원문, 별표(행위제한·주차·부담금), 별지서식 다운로드 |

## 주요 기능

- **work_type 자동판별** — 건축물대장을 조회해 빈 땅인지 기존 건물인지 사실 확인 후 신축/용도변경/대수선을 판정(코드가 신축으로 가정하지 않음). 용도변경이면 층별개요로 바꾸려는 층의 현재 용도까지 확인.
- **다차원 종합판정** — 이 사안에서 실제로 가부를 가른 축들(용도·접도·권원·조례·의제 등)을 LLM이 골라 각각 충족/주의/확인필요/불가로 합성.
- **fail-closed 안전판정** — 맹지(접도의무 미해소)·강한 행위제한 규제가 근거 없이 남으면 거짓 '가능'을 막고 '확인필요+사람검토'로 보류.
- **제출서류 전수확보** — 시행규칙 조문의 첨부서류를 호·목 단위로 라이브 추출. 의제(농지전용·산지전용·개발행위 등) 서류는 건축허가 신청 패키지에 "함께 첨부"로 병합. 조건부 서류는 케이스로 해당/비해당/확인필요 판정.
- **규모·부담금** — 건폐율·용적률 상한 가늠, 부설주차 소요대수, 농지보전부담금 등. 율·단가는 모두 법령 원문에서 읽은 값만 사용.

## 스택

- **백엔드**: FastAPI + LangGraph(ReAct 에이전트) + SSE 실시간 스트리밍
- **프론트**: 바닐라 JS(실시간 실행 트레이스 + 진단 카드, same-origin 서빙)
- **RAG(선택)**: 자치법규 별표 사전 인덱스(pgvector)

## 설치 & 실행

[uv](https://docs.astral.sh/uv/) 사용을 권장합니다.

```bash
# 1) 의존성
uv sync

# 2) 환경변수 — 예시를 복사해 키를 채웁니다(발급처는 .env.example 주석 참고)
cp .env.example .env
#   VWORLD_KEY, DATAGO_KEY, LAW_OC, KAKAO_KEY 등

# 3) 백엔드 실행(프론트 same-origin 서빙)
uv run uvicorn app:app --app-dir backend --port 8000
```

브라우저에서 http://127.0.0.1:8000 접속. (빠른 검증은 `FORCE_STUB=1`로 결정적 stub 경로 실행.)

## 환경변수

모든 API 키는 `.env`(gitignored)에만 둡니다 — 코드·커밋에 노출 금지. 필요한 변수와 발급처는 [`.env.example`](.env.example)을 참고하세요.

## 프로젝트 구조

```
backend/    FastAPI 앱 · SSE 래퍼 · 상태 초기화
agent/      LangGraph 그래프 · 도구 · 상태 · 시스템 프롬프트

```

## 정직 범위 (한계)

거짓 완료·과대표기를 피하기 위해 현재 한계를 명시합니다.

- **조례 RAG는 전국이 아님** — 경기 남부 + 춘천 등 일부 시군의 도시계획조례 용도지역 별표만 사전 인덱싱했습니다. 그 외 지자체/용도지역은 라이브 폴백 또는 '확인필요'로 처리합니다.
- **도시계획조례 중심** — 건축·주차장·경관 조례 별표는 미구현(해당 시 '확인필요'로 정직 표기).
- **부담금 단가 데이터원 없음** — 금액은 율·공시지가 기반 가늠이며, 단가 출처가 없으면 '확인필요'.
- **상위 데이터 한계** — 일부 산지(산 번지) 주소는 지오코더가 해석하지 못할 수 있습니다(이 경우 날조하지 않고 재입력을 요청).
- **stub 모드 판정은 보수적** — `FORCE_STUB`는 결정적 스캐폴드로, 실제 멀티홉 호목 해소·다차원 판정은 LLM 모드에서만.
- 본 결과는 **사전 가늠**이며 법적 효력이 없습니다. 실제 인허가는 관할 행정청·건축사 검토가 필요합니다.
